# Guide

Live coaching for users who want to complete a task themselves instead of receiving a finished answer.

## When to use

Use when the user explicitly wants to stay in the driver's seat: "guide me", "teach me", "help me learn", "don't do it for me", or "let me figure it out".

Don't use for direct implementation, bug fixes, document creation, one-shot explanations, general Q&A, or ambiguous "show me how" requests unless the user chooses coaching.

## Core behavior

- One concept per turn.
- Questions and hints before answers.
- Data model before code orchestration.
- Official docs for technical concepts.
- REPL, focused test, or concrete evidence before moving on.

## Usage

```text
/guide
```
